    for everyone,  who wants to try example in mysql:
    edit file
    application.ini
    remove:
    resources.db.adapter = "PDO_SQLITE"
    resources.db.params.dbname = APPLICATION_PATH "/../data/db/guestbook.db"

    add and configure
    resources.db.adapter = "PDO_MYSQL"
    resources.db.params.host = "localhost"
    resources.db.params.username = "username"
    resources.db.params.password = "password"
    resources.db.params.dbname = "zf"

    in mysql create database zf
    and execute sql:

    CREATE TABLE IF NOT EXISTS `guestbook` (
      `id` int(11) NOT NULL AUTO_INCREMENT,
      `email` varchar(32) NOT NULL DEFAULT 'noemail@test.com',
      `comment` text,
      `created` datetime NOT NULL,
      PRIMARY KEY (`id`)
    ) ENGINE=MyISAM  DEFAULT CHARSET=latin1 AUTO_INCREMENT=3 ;

    --
    --  `guestbook`
    --

    INSERT INTO `guestbook` (`id`, `email`, `comment`, `created`) VALUES
    (1, 'ralph.schindler@zend.com', 'Hello! Hope you enjoy this sample zf application!', '2012-03-22 00:00:00'),
    (2, 'foo@bar.com', 'Baz baz baz, baz baz Baz baz baz - baz baz baz.', '2012-03-20 00:00:00');

                                Stryks 3 years ago

                                Despite the negative feedback, this is actually a great introduction to the basic concepts of models and model interaction in ZF.

                                Where it fails (quite badly I feel) is in introducing too much to achieve something quite simple.

                                In my opinion:

                                - The command line database configuration should be removed; just showing and explaining the ini file should be enough.  The CLI stuff at this stage obscures rather than clarifies.

                                - Drop SQLite in preference of mySQL. SQLite has it's benefits, but I cant think of any packaged installs I have seen anywhere which did not include an option for mySQL.  A self install does of course, but then if you have the nonce to do a manual install, then you're minutes away from having mySQL installed as well; just like the majority of dev environments (ok - I assume the majority).  

                                - With the DB changed over to mysql, simply provide a download of the SQL file.  Give a brief command line or recommend a DB tool by all means, but that huge slab of unexplained code across three files in an easily misconfigured folder has got to go.

                                I feel the quickstart should embody best practices, touching on the most common components, while also passing over peripheral complexities.  People starting with ZF need to see how to interact with the model, not how to knock up a single-use db creation script.  

                                It seems problematic to keep pointing newcomers to ZF at this tutorial knowing that they will get to this point and then balk at the code (thinking that the slab is required for all models) and frequently grind to a halt trying to get it to work, while the core points in this page show a really excellent modelling structure.

                                This page could be reduced by as much as a third and still impart as much information, which I think would make the information that IS here much more visible, accessible and useful.

                                But as I said, just my opinion.

                                                                            1) The part where it says: "First we need to initialize our Db resource. As with the Layout and View resource, we can provide configuration for the Db resource. We can do this with the zf configure db-adapter command:"

                                                                            This didn't work for me in Windows command line because of the quotes. I needed to switch double and single quotes around.

                                                                            zf configure db-adapter "adapter=PDO_SQLITE&dbname=APPLICATION_PATH '/../data/db/guestbook.db'" production

                                                                            instead of

                                                                            zf configure db-adapter 'adapter=PDO_SQLITE&dbname=APPLICATION_PATH "/../data/db/guestbook.db"' production
